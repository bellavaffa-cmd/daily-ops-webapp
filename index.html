Slack Notifications

Get a Slack message every time a packing error is submitted. You'll need to create an Incoming Webhook in your Slack workspace — no coding required.

  1. Create a Slack App.
    Go to api.slack.com/apps and click Create New App → From scratch. Give it a name (e.g. Packing Errors) and choose your workspace.
  2. Enable Incoming Webhooks.
    In your app settings, go to Incoming Webhooks and toggle it On. Scroll down and click Add New Webhook to Workspace. Choose the channel where errors should be posted, then click Allow.
  3. Copy the Webhook URL.
    After authorising, a Webhook URL will appear on the page — it starts with https://hooks.slack.com/services/…. Copy it and paste it below.

Make Available Online (GitHub Pages)

Host this webapp at a free public URL using GitHub Pages — no server required. Anyone with the link can open it from any device.

  1. Create a GitHub account.
    Go to github.com and sign up (free). If you already have an account, sign in.
  2. Create a new public repository.
    Click + → New repository. Give it a name (e.g. daily-ops-webapp). Set visibility to Public, then click Create repository.
  3. Upload this file renamed as index.html.
    On the repo page, click uploading an existing file. Rename packing-error-report.html to index.html before uploading — GitHub Pages serves index.html as the default page. Click Commit changes.
  4. Enable GitHub Pages via GitHub Actions.
    Go to Settings → Pages. Under Source, open the dropdown and select GitHub Actions. Two templates appear — click Configure on Static HTML. Click Commit changes to save the workflow.
  5. Wait ~30 s, then open your site.
    GitHub runs the workflow automatically. Once it shows a green checkmark in the Actions tab, your site is live at:
    https://<your-username>.github.io/<repo-name>
💡 Tip: Bookmark the URL and share it with your team. Everyone who opens it will always get the latest version of the webapp.
