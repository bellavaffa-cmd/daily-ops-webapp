WMS → Report Error scan

This bookmarklet reads the order currently loaded on a wms.golocad.com Packing Station screen (Order Code, Tote Number, Warehouse, Brand, SKU) and opens the Daily Ops Report an Issue form with Order ID, Tote Number, Warehouse, and Brand already filled in.

Step 1 — Install: drag this button to your browser's bookmarks bar.

📋 Scan WMS → Report Error

If your bookmarks bar is hidden, show it first (Chrome: ⌘⇧B / Ctrl+Shift+B), then drag the button there.

Step 2 — Use it:

  1. Open a Packing Station order on wms.golocad.com so an order is loaded on screen.
  2. Click the Scan WMS → Report Error bookmark.
  3. A new tab opens the Daily Ops app with the Report an Issue form already open — Order ID, Tote Number, Brand, and (if recognised) Warehouse are filled in.
  4. Pick the Error Type, add any notes/photos, and submit as usual.
Notes
  • Warehouse is matched by a fuzzy text match against the live warehouse list. If it can't find a confident match, the field is left blank — just pick it manually.
  • Brand is read from the first "[Bracketed]" segment in Packing Instructions (e.g. [Nuffield Pharm]). If an order has no bracketed brand there, the field is left blank.
  • Nothing is sent anywhere by this bookmarklet — it only reads the current WMS page and opens the Daily Ops app in a new tab with the data in the URL. No report is submitted until you press Submit yourself.
  • SKU / Qty / Pack Type are also captured from the scan, but the current Report form's error types (AWB Error, Unscannable Stock, Printer Issue, Camera Issue, Other) don't have a place to show them yet — that data isn't discarded, just unused for now.
